Publish your Angular app to Heroku

on node, angular, heroku

Heroku is great for hosting your development projects (particularly in-progress). In fact, I recommend publishing your project sooner rather than later so that you don't have any surprises in deployment later on.

Getting Heroku to serve your Angular application is pretty simple but does require a little bit of configuration. Moreover, these same steps could work for any front-end project, not just Angular.

This article assumes:
You have a Heroku account
You have the Heroku Toolbelt installed on your machine

Add Express

Firstly, you'll want to npm install --save express gzippo in your project directory.

Express is a node-based web server which Heroku will use to serve your application files.

Gzippo allows you to compress (gzip) the files before they are sent on the wire to reduce the size, improving performance.

Create an Express server file

Create a file named web.js in your project root. In this file, add the following lines:

var gzippo = require('gzippo');
var express = require('express');
var morgan = require('morgan');
var app = express();

app.use(morgan('dev'));
app.use(gzippo.staticGzip("" + __dirname + "/dist"));

app.route('/*').get( function(req, res) {
    return res.sendFile(__dirname + "/dist/index.html");
});

app.listen(process.env.PORT || 5000);

This file includes necessary dependencies, and configures Express to respond to all requests with the file at the path in the res.sendFile method. You will need to update this path according to where your final application bundle resides.

Add Procfile and Update package.json

Heroku relies on a file called Procfile to know how to start your application. Note that Procfile has no file extension.
To this file, you just need to add web: node web.js. Web lets Heroku know that you want to serve HTTP content, and the remainder is command to invoke node with the Express file you created in the previous step.

Push to Heroku

Before you push to Heroku, it's always a good idea to run heroku local to make sure your configuration is correct. If everything looks good then you can git push heroku master. If that completes successfully you are good to go!
