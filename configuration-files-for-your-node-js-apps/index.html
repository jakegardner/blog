Configuration files for your Node.js apps

on node

For any Node.js application that interacts with external resources (APIs, databases, etc), you will likely need some environment-specific configuration. Now, hopefully you are not hard-coding these values!

            // BAD! 
            mongodb.hostname = "databaseserver";
            mongodb.port = "27017";
            mongodb.database = "mydatabase";
            ...

Storing these values in an external file makes it easy to change environments without having to modify the code. Fortunately, require makes it easy to load values from .js or .json files.

            // Better!
            var config = require('./dbconfig.js');

            mongodb.hostname = config.hostname;
            mongodb.port = config.port;
            mongodb.database = config.database;
            ...

Require will parse either .js or .json files and return an object you can use in your code.

Why choose .js or .json? I prefer to use .js for configuration while only use .json for data (for example, test fixtures). However, both formats achieve the same result.

.js format
            exports.hostname = "myserver";
            exports.port = "27017";
            exports.database = "mydatabase";
.json format
            {
              "hostname": "myserver",
              "port": "27017",
              "database": "mydatabase"
            }

Knowing how easy this is to use, hope you will make use of external configuration files in your Node.js apps!
