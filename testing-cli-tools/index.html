Testing CLI Tools

on node, testing

While I was developing the s3syncr command-line (CLI) utility, I discovered I needed a way to test the function of the program as a whole. There are separate tests for the parsing of input to the program, but I wanted to verify the interaction of all the components as well as the output.

Since console output is the only interface given to the user in a CLI tool, consistent output is critical.

So, I made use of Node to run the utility inside a Mocha test.

Node exec()

describe("No parameters", function() {
    it("should output No operation", function(done) {
        require('child_process').exec("node index.js",  
            function(err, stdout, stderr) {
              assert(err);
              assert.strictEqual(stderr, "No operation\n");
              done();  
        });
    });
});

This test uses the Node-native child_process.exec method which runs a shell command and gives you access to what is written to console.log() and console.error() through the stdout and stderr parameters, respectively. You can also pass in an options object to specify current working directory, environment variables and so on.

Asserting Output

assert.strictEqual(stderr, "No operation\n");

Now that console output is available, to execute tests you just need to assert on stdout or stderr string content. 

assert(err);

You can also use the err parameter to check for uncaught exceptions.

Automating Input

Using exec() in the before and after test hooks to handle varying test input gives you a path towards comprehensive automated testing!

For example, you can have some tests that create a file, run the test, and then remove the file afterwards.

beforeEach(function(done) {
    require('child_process').exec("echo 'test' > test.txt", 
        function(err, stdout, stderr) {
            done();
    });
});

afterEach(function(done) {
    require('child_process').exec("rm test.txt", 
        function(err, stdout, stderr) {
            done();
    });
});

Wrap Up

If you write any kind of CLI, this is a simple way to verify output.

Honestly, this kind of test could be done in any scripting language, but using Node and Mocha allowed me to integrate with the rest of the testing suite.
